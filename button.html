Usage

Primary Button Usage Guidelines
  • 1) Use a primary button to indicate the primary action, i.e. what we want the user to click or experience.
  • 2) The button should have text or icon that should describe the action of the button.
  • 3) Don't use a button to navigate to another place, Use tertiary button instead.
  • 4) Use sentence-style capitalization Ex: Submit Save.
  • 5) Color of the Button should be as per design system.

Usage

Floating Action Button Usage Guidelines
  • 1) Use Floating Action Button for most commonly used action on the screen.
  • 2) Use descriptive icons for FAB, avoid using unclear icons. Examples-.
  • 3) Use only one FAB per screen.
  • 4) When we have set of actions inside FAB, those action should be related to primary action. Add labels to those actions.

Do

Don't

Represent the most common primary action with a floating action button, such as the primary set of tools.

The floating action button cannot be represented with inverted texts which effects readability.

A primary action can have minimum of three options and maximum of five options.

A primary action shouldnt contain a single action alone.

Use Floating Action Button for primary, positive actions.

Do not use Floating Action Button for minor, overflow, unclear, or destructive actions.

Best Practices

Accessibility:

  • 1) Button can be triggered by clicking left click on the mouse and by pressing enter and spacebar in the keyboard.
  • 2) Differentiating button on different states. Ex: what happens on selecting the button, (Hovering, Active state, Focusing).
  • 3) Make text of the button where the user can easily understand the action which is going to be taken. Ex: Use Submit rather than Ok.
  • 4) Touch target of the button should be consistent in way that the user can use the button.
  • 5) Ex: Use the spacing consistent for icon buttons when we have more actions next to each other.

Best Practices:

  • 1) Wherever the component is used, always place the primary button on the right and the secondary button to the left of it.
  • 2) The text and the icon should be aligned in center of the button.
  • 3) Use only one primary button to a screen. Using multiple primary buttons will lead the user to confuse about action.
  • 4) Primary, secondary, and tertiary buttons should be differentiated visually.
  • 5) Pay attention on the size of the button when it scale to smaller devices like tablet, smartphone devices.
